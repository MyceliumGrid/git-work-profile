# 更新日志

## v2.1.0 - 多语言支持 (2024-11-30)

### 🌍 新功能
- **完整的多语言支持**: 实现中英文双语界面
  - 所有用户界面文本支持中英文切换
  - 命令行参数描述多语言化
  - 交互式界面多语言化
  - 报告输出多语言化
  - 错误和提示信息多语言化
  - 所有 `fmt.Errorf` 错误消息多语言化
- **语言切换**: 通过 `GIT_PROFILE_LANG` 环境变量控制语言
  - 默认中文 (zh)
  - 支持英文 (en)
- **智能语言检测**: 从环境变量自动加载语言设置

### 🔧 改进
- 优化命令行帮助信息的多语言显示
- 统一所有模块的多语言支持
- 移除所有硬编码的文本字符串
- 修复 `fmt.Errorf` 格式字符串必须为常量的问题
- 完善错误消息的多语言覆盖

### 📚 文档
- 创建完整的中英文双语文档
  - `README.md` (中文) 和 `README_EN.md` (英文)
  - `QUICKSTART.md` (中文) 和 `QUICKSTART_EN.md` (英文)
- 添加文档间的语言切换链接
- 更新 README.md 添加多语言使用说明
- 更新 QUICKSTART.md 添加语言设置说明

## v2.0.0 - 重大改造 (2024-11-30)

### 🎉 重大变更
- **完全重构**: 从"工作报告生成器"改造为"开发者画像分析器"
- **新的定位**: 专注于开发者画像、项目经验和技术栈分析

### ✨ 新功能
- **交互式模式**: 新增友好的交互式配置界面，无需记住命令行参数 🎨
  - 自动检测和提示输入API密钥
  - 引导式选择分析类型、时间范围、仓库模式
  - 智能默认值和路径建议
  - 配置摘要和确认
- **开发者画像分析**: 全面分析技术栈、工作风格、专业领域和核心竞争力
- **项目经验总结**: 智能总结项目类型、技术实践和工程能力
- **技术栈分析**: 深度分析编程语言、框架、工具链和技术广度
- **多维度统计**: 提交数、仓库数、文件类型分布等详细统计
- **JSON输出**: 新增JSON格式输出，便于程序处理

### 🔄 变更
- 默认分析时间范围从"过去7天"改为"最近6个月"
- 时间范围选项从 `day/week/month/year` 改为 `3m/6m/1y/2y`
- 默认输出格式从 `text` 改为 `markdown`
- 移除了 `--date` 参数（使用 `--from` 和 `--to` 代替）
- 移除了 `--prompt` 参数（使用 `--analysis` 代替）
- 命令目录从 `cmd/git-work-log` 改为 `cmd/git-work-profile`

### 📝 提示词模板
- 新增 `prompts/developer-profile.txt` - 开发者画像分析
- 新增 `prompts/project-experience.txt` - 项目经验总结
- 新增 `prompts/techstack-analysis.txt` - 技术栈分析
- 移除旧的 `basic.txt`, `detailed.txt`, `targeted.txt`

### 🗑️ 移除功能
- 移除了日报、周报、月报、年报的概念
- 移除了自定义提示词文件功能（可能在未来版本恢复）

### 📚 文档
- 完全重写 README.md
- 新增 EXAMPLES.md 示例文档
- 新增 QUICKSTART.md 快速开始指南
- 新增 INTERACTIVE_MODE.md 交互式模式详细说明
- 新增 MIGRATION.md 迁移指南
- 新增 IMPLEMENTATION_CHECKLIST.md 实施检查清单
- 新增 CHANGELOG.md 更新日志

### 🐛 修复
- 修复了多仓库分析时的统计问题
- 优化了文件类型识别逻辑

### 📦 依赖
